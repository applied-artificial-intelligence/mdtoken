### Pre-commit Integration (IMPORTANT - Open Question)
**User's Question**: "how does the project integrate with pre-commit? should it be called pre-commit-mdtoken-count?"

**Current Implementation**:
- Hook ID: `markdown-token-limit` (in `.pre-commit-hooks.yaml`)
- Entry point: `mdtoken check`
- Repository name: `mdtoken` (not prefixed with `pre-commit-`)

**Hook Configuration**:
```yaml
# .pre-commit-hooks.yaml
- id: markdown-token-limit
  name: Markdown Token Limit
  entry: mdtoken check
  language: python
  files: \.md$
  pass_filenames: true
```

**User Usage**:
```yaml
# In user's .pre-commit-config.yaml
repos:
  - repo: https://github.com/applied-artificial-intelligence/mdtoken
    rev: v1.0.0
    hooks:
      - id: markdown-token-limit
```

**Naming Convention Analysis**:
- ✅ Current approach follows standard pre-commit pattern (e.g., `black`, `flake8`, `mypy`)
- ✅ Repository name is the tool name (`mdtoken`)
- ✅ Hook ID describes what it does (`markdown-token-limit`)
- ❌ NOT using `pre-commit-` prefix (which is older convention)

**Modern Pre-commit Pattern** (recommended):
- Repository: `org/tool-name` (e.g., `psf/black`, `pre-commit/pre-commit-hooks`)
- Hook ID: descriptive name (e.g., `black`, `trailing-whitespace`)
- Entry: tool command (e.g., `black`, `mdtoken check`)

**Current implementation is CORRECT** - no changes needed unless user prefers different naming.
